REST API for employee attendance tracking with ESP32 integration support.

Smart Automatic Endpoints

These endpoints automatically detect whether to start or end a shift based on the employee's current status.

Automatically Start/End Shift (by ID)
POST
/api/shift

Automatically starts a new shift or ends the current active shift based on employee ID.

Request Body:
{ "employee_id": 1, "work_location": "Zoo" // Required only when starting }
Response (Starting Shift):
{ "success": true, "action": "started", "attendance_id": 15, "employee_name": "Janko Mrkvička", "work_location": "Zoo", "start_time": "14:30", "message": "Shift started successfully" }
Response (Ending Shift):
{ "success": true, "action": "ended", "attendance_id": 15, "employee_name": "Janko Mrkvička", "hours_worked": 4.5, "start_time": "14:30", "end_time": "19:00", "message": "Shift ended successfully" }
ESP32 Example:
// For both clock in and clock out: http.postJson("https://your-domain.com/api/shift", { "employee_id": 1, "work_location": "Zoo" // Include for clock in, omit for clock out });
Automatically Start/End Shift (by Name)
POST
/api/shift_by_name

Automatically starts a new shift or ends the current active shift based on employee name.

Request Body:
{ "employee_name": "Janko Mrkvička", "work_location": "Zoo" // Required only when starting }
Name Matching Rules:
  • Full name: "Janko Mrkvička" → exact name+surname match
  • Partial name: "Janko" → searches in name and surname fields
  • Case insensitive: "janko mrkvička" works the same
Response (Starting Shift):
{ "success": true, "action": "started", "attendance_id": 15, "employee_id": 1, "employee_name": "Janko Mrkvička", "work_location": "Zoo", "start_time": "14:30", "message": "Shift started successfully" }
ESP32 Example:
// Simple usage for both operations: http.postJson("https://your-domain.com/api/shift_by_name", { "employee_name": "Janko Mrkvička", "work_location": "Zoo" // Omit this field to clock out });

Manual Control Endpoints

Use these endpoints when you need explicit control over starting or ending shifts.

Start Shift (by ID)
POST
/api/start_shift

Explicitly start a new shift using employee ID.

Request Body:
{ "employee_id": 1, "work_location": "Zoo" }
Response:
{ "success": true, "attendance_id": 15, "employee_name": "Janko Mrkvička", "message": "Shift started successfully" }
Start Shift (by Name)
POST
/api/start_shift_by_name

Explicitly start a new shift using employee name.

Request Body:
{ "employee_name": "Janko Mrkvička", "work_location": "Zoo" }
Response:
{ "success": true, "attendance_id": 15, "employee_id": 1, "employee_name": "Janko Mrkvička", "message": "Shift started successfully" }
End Shift (by ID)
POST
/api/end_shift

Explicitly end a shift using employee ID or attendance ID.

Request Body (Option 1 - by attendance_id):
{ "attendance_id": 15 }
Request Body (Option 2 - by employee_id):
{ "employee_id": 1 }
Response:
{ "success": true, "hours_worked": 4.5, "employee_name": "Janko Mrkvička", "message": "Shift ended successfully" }
End Shift (by Name)
POST
/api/end_shift_by_name

Explicitly end a shift using employee name.

Request Body:
{ "employee_name": "Janko Mrkvička" }
Response:
{ "success": true, "hours_worked": 4.5, "employee_name": "Janko Mrkvička", "message": "Shift ended successfully" }

Error Handling

All API endpoints return consistent error responses:

{ "error": "Error message description" }
Common HTTP Status Codes:
  • 200 - Success
  • 400 - Bad Request (missing or invalid parameters)
  • 404 - Not Found (employee or attendance record not found)
  • 500 - Internal Server Error

ESP32 Integration Tips

Recommended Endpoints:
  • Use /api/shift_by_name for simplest integration
  • Use /api/shift if you have employee IDs stored
Example Arduino Code:
#include <HTTPClient.h> #include <ArduinoJson.h> void clockInOut(String employeeName, String location = "") { HTTPClient http; http.begin("https://your-domain.com/api/shift_by_name"); http.addHeader("Content-Type", "application/json"); DynamicJsonDocument doc(1024); doc["employee_name"] = employeeName; if (location != "") { doc["work_location"] = location; } String jsonString; serializeJson(doc, jsonString); int httpCode = http.POST(jsonString); if (httpCode == 200) { String response = http.getString(); // Parse response JSON DynamicJsonDocument resDoc(1024); deserializeJson(resDoc, response); String action = resDoc["action"]; Serial.println("Shift " + action + " successfully"); } http.end(); }
